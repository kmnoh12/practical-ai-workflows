## 3. 30-minute test

1. Choose one real sample input.
2. Run the smallest possible workflow.
3. Save the output.
4. Check the result against the original source or expected user action.
5. Record what failed or needed manual correction.

## 4. Keep / revise / stop

- Keep if the workflow saves time and is easy to check.
- Revise if the output is useful but too fragile.
- Stop if it creates more review work than manual execution.
